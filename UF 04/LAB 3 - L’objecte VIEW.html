LAB 3 - L’objecte VIEW
Les vistes són taules virtuals que ens permeten facilitar certes consultes que solen repetir-se moltes vegades.
Imaginem que de la nostra taula teachers només volem veure una vista amb el nom i cognom junts
Fem una query nova:
create view vTeachers as select id, name || ' ' || surname as professor from dades.teachers
Copy
SQL
Amb això hem creat un nou objecte de la base de dades, concretament una vista.
Si ara volem veure el resultat, no cal fer tota la query complerta cada cop, sinó que senzillament escrivim:
select * from dades.vTeachers
SQL
O com que vTeachers és un objecte (objecte vista) podem navegar pel llistat d’objectes de la nostra base de dades i també el trobarem:
Les dades d’una vista no es desen de forma redundant a la base de dades. La vista realment és la sentència select * from dades.vTeachers, que a la vegada fa una consulta a la taula dades.teachers en aquest cas. És a dir si inserim un nou professor a la base de dades la vista s’actualitza automàticament.
D’objectes vista n’hi ha molts. Alguns els creem nosaltres i d’altres ja estan creats pel sistema. Si vols veure’ls tots pots ecriure la comanda:
select * from information_schema.views
SQL
En aquest cas no ho veuràs a l’arbre d’objectes, perquè pgAdmin ho amaga al ser objectes del propi sistema.
Les vistes també es poden comportar com a taules. Anem a fer un exemple. Crearem uns quants alumnes a la nostra taula students:
insert into students values (1, 'Joan', 'Badia'); insert into students values (2, 'Maria', 'Montbui'); insert into students values (3, 'Anna', 'Vidal');
SQL
Ara crearem una taula anomenada studentsToTeachers on desarem quins alumnes pertanyen a cada professor:
create table studentsToTeachers ( id serial, student_id integer, teacher_id integer, PRIMARY KEY (id), FOREIGN KEY (student_id) REFERENCES public.students (id), FOREIGN KEY (teacher_id) REFERENCES dades.teachers (id) )
SQL
Fixa’t que el tipus de dada que coneixies com a AUTO_INCREMENT en MySQL aquí es diu SERIAL.
I farem que el professor 1 (Anton Dalmau) tingui els alumnes 1 i 2 (Joan Badia i Maria Montbui) i que el professor 2 (Eduard Sorita) tingui els alumnes 1 i 3 (Joan Badia i Anna Vidal).
INSERT INTO public.studentstoteachers(student_id, teacher_id) VALUES (1, 1); INSERT INTO public.studentstoteachers(student_id, teacher_id) VALUES (2, 1); INSERT INTO public.studentstoteachers(student_id, teacher_id) VALUES (1, 2); INSERT INTO public.studentstoteachers(student_id, teacher_id) VALUES (3, 2);
SQL
Ara voldrem mostrar quins alumnes té cada professor fent un inner join entre la taula studentsToTeachers i la vista vTeachers
select * from dades.vTeachers TE inner join studentsToTeachers STT on TE.id = STT.teacher_id
SQL
Aquí veiem com combinem taules i vistes.
Si volem, per exemple, que no es repeteixi l’identificador del professor i que no surti tampoc el ID de la taula studentsToTeachers, podem fer els següents filtres:
select * from dades.vTeachers TE inner join studentsToTeachers STT on TE.id = STT.teacher_id
SQL
I ja per acabar de reblar el clau podríem voler només mostrar els noms del professors amb el nom dels alumnes:
select TE.professor, ST.name || ' ' || ST.surname from dades.vTeachers TE inner join studentsToTeachers STT on TE.id = STT.teacher_id inner join students ST on ST.id = STT.student_id
SQL
Aquesta consulta ja és una mica complexa oi? I si l’hem d’escriure cada vegada és molt pesat. Què podem fer…? Exacte! Una vista!
create view dades.vTeachersAndStudents as select TE.professor, ST.name || ' ' || ST.surname as alumne from dades.vTeachers TE inner join studentsToTeachers STT on TE.id = STT.teacher_id inner join students ST on ST.id = STT.student_id
SQL
