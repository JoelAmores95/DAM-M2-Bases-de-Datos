Schemas
Si t’hi fixes, quan has instal·lat pgAdmin i l’has connectat al SGBD, per defecte tens dues bases de dades: la pròpia de postgres i la de l’usuari inicial (root en el nostre exemple). I dins de cada base de dades un munt de subapartats:
El que veurem ara són els Schemas que bàsicament són els objectes que pot contenir una base de dades PostgreSQL. I primer veurem com funciona un tipus d’objecte que ja coneixem: les taules.
Una nova base de dades
Crearem una nova base de dades per fer proves. Per fer-ho anirem combinant el treball amb l’eina gràfica, però també ordres que es poden utilitzar des d’una CLI per si mai has de gestionar d’aquesta manera un servidor de base de dades.
En col·locarem sobre alguna de les dues bases de dades que tenim i anirem a la pestanya “Tools” i escollirem “Query Tool”. Se’ns obrirà una finestra nova i escriurem:
create database Institut
SQL
Tot seguit polsarem el símbol de Play:
I ja tenim la nova base de dades creada. Fixa’t que normalitza els noms i canvia les majúscules per minúscules:
Seleccionant la base de dades “institut” veiem que per defecte tenim el schema public. Tot i que podem treballar amb ell, en crearem un de nou.
Ho farem també amb comandes, per tant utilitza el “Query Tool”
create schema dades
SQL
Taules
Els schema de Postgre realment són espais de noms per agrupar objectes i funcionalitats. Per provar-ho crearem una taula de professors a l’schema dades i una taula d’alumnes a l’schema public:
create table dades.teachers( id int, name varchar, surname varchar, salary float, PRIMARY KEY (id) )
SQL
create table students( id int, name varchar, surname varchar, PRIMARY KEY (id) )
SQL
Fixa’t que si vull crear una taula a l’schema public no cal posar public.students.
Amb l’eina gràfica puc escriure totes les comandes juntes i seleccionar la que vulgui executar i clicar sobre “Play”
Inserir i consultar dades
Anem a inserir algunes dades i a veure diferents formes de fer-ho. Continuarem utilitzant comandes:
insert into dades.teachers values (1, 'Anton', 'Dalmau', 1890.95) insert into dades.teachers(salary, id, surname, name) values (1955.26, 2, 'Sorita', 'Eduard')
SQL
Aquí veiem dues maneres d’inserir dades. Si aquestes estan ordenades en el format de les columnes de la taula no cal posar els títols. En cas contrari sí.
Si volem consultar les dades cal fer un senzill:
select * from dades.teachers
SQL
Actualitzar dades
És tant senzill com fer:
update dades.teachers set salary = 2000 where id = 1
SQL
